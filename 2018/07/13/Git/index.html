让Git来高效管理你的项目

发表于 2018-07-13
字数统计: 909 | 阅读时长 ≈ 3

1.Git介绍

　　Git是一个开源的分布式版本控制系统（无联网也能正常工作），用于敏捷高效地处理任何或小或大的项目。假如现在你有一个文件夹，里面可以是项目，也可以是你的个人笔记，或者是你的简历、毕业设计等等，都可以使用git来高效管理。

2.安装Git

在Windows上安装Git

从https://git-for-windows.github.io 下载，然后按默认选项安装即可，安装完成后，在桌面右键就可以看到多了Git Gui Here和Git Bash Here两个选项了。

3.配置Git

安装完成后，还需要配置用户名和密码，打开git-bash.exe，在命令行输入：

  • $ git config –global “Your Name”
  • $ git config –global “email@example.com“

“Your Name”： 是每次提交时所显示的用户名，因为Git是分布式版本控制系统，当我们push到远端时，就需要区分每个提交记录具体是谁提交的，这个”Your Name”就是最好的区分。

“email@example.com“： 是你远端仓库的email

–global：用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然我们也可以对某个仓库指定不同的用户名和Email地址。

4.远程仓库：

github：请自行注册GitHub账号
由于你的本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，所以，需要一点设置：

创建SSH Key。在用户主目录下，看看有没有.ssh目录，如果有，再看看这个目录下有没有id_rsa和id_rsa.pub这两个文件，如果已经有了，可直接跳到下一步。如果没有,打开Git Bash，创建SSH Key：$ ssh-keygen -t rsa -C “你的邮箱地址”

登陆GitHub，打开“Account settings”，“SSH Keys”页面：然后，点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容：

当然，GitHub允许你添加多个Key。假定你有若干电脑，你一会儿在公司提交，一会儿在家里提交，只要把每台电脑的Key都添加到GitHub，就可以在每台电脑上往GitHub推送了。

5.创建远程库且和本地库关联

1，在GitHub上创建一个新仓库；

2，把本地仓库与GitHub进行关联:git remote add origin git@github.com:your usename/your Repo.git

备注：当你第一次使用git的clone或push命令连接GitHub时,会得到一个警告，这是因为Git使用SSH连接，而SSH连接在第一次验证GitHub服务器的Key时，需要你确认GitHub的Key的指纹信息是否真的来自GitHub的服务器，输入yes回车即可。

6.开始使用

在你要管理的目标文件夹下右键点击Git Bash Here，输入命令：

  • git init （创建.git文件）

就会创建一个 .git 隐藏文件，相当于已经建立了一个本地仓库。
.git是用来跟踪管理版本库的，最好不要手动修改删除。

  • git add . （全部添加到暂存区）
  • git commit -m ‘ 提交信息 ‘ （提交暂存区的记录到本地仓库）
  • git push -u origin master (提交到远程仓库)
    首次推动到远程库需要添加-u，以后就不必了

7.其它命令

  • git rm 文件名 //删除文件
  • git rm -r 文件夹名 //删除文件夹
  • git diff //查看具体的修改
  • ssh -t git@github.com //测试是否绑定了github
  • git status //查看本地修改了多少文件
  • git log //查看commit的版本号
  • git reset –hard <版本号> //撤回到需要的版本
  • git push origin master – force //重新推送
